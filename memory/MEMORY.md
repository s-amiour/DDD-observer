# Project Memory

## Project
TypeScript DDD exercise — restaurant/e-commerce domain. Entry point: `index.ts`.

## Key Context
- `index.ts` is a **showcase file** — intentionally error-prone code is by design (Phase 2 "feel the pain" stage)
- Do NOT flag deliberately broken patterns as bugs during reviews unless they deviate from the intended showcase
- Only flag things that are unintentionally wrong or stray from what the phase is trying to demonstrate

## Exercise Phases
- Phase 1–2: Plain dumb objects, silent bugs on purpose
- Phase 3: Branded types
- Phase 4: Smart constructors
- Phase 5: Value objects
- Phase 6: Entities with uuid
- Phase 7: Observer pattern

## Agent Rules
- AI agents must NOT write code — only docs/markdown files
- Always prompt user before writing or committing any file
